Common issues & troubleshooting
---------------------------------
- Horizontal/X scrollbar on some pages:
	- Cause: flex children with non-shrinking `min-width` values (e.g. `min-w-[360px]`) or use of `w-screen` with padding. These can cause the layout to exceed the viewport width.
	- Quick fix: allow flex children to shrink by adding `min-w-0` on flex items and use responsive min-widths, e.g. `className="flex-1 min-w-0 md:min-w-[360px]"`.
	- Debugging tip: run this in the browser console to find elements wider than the viewport:
		```js
		Array.from(document.querySelectorAll('*'))
			.filter(el => el.scrollWidth > document.documentElement.clientWidth)
			.forEach(el => console.log(el, el.scrollWidth, el.getBoundingClientRect()));
		```

- Git push to remote failing with `Repository not found`:
	- Ensure the remote repository exists and your account has access.
	- If private, authenticate with GitHub (PAT or SSH) before pushing.

Coding & style notes
---------------------------------
- Tailwind patterns used extensively. Prefer `min-w-0` on flex children when you want inputs or text to shrink.
- Avoid `w-screen` on elements that also have `px-*` padding; use `w-full` inside a padded container.
